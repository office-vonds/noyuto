# 実行手順

## 1. Notionからタスク取得
mcp__claude_ai_Notion__notion-search を使い、以下を順に検索:
- "TODO" で検索
- "残課題" で検索  
- "タスク" で検索
- "進行中" で検索

取得したページそれぞれについて mcp__claude_ai_Notion__notion-fetch でページ内容を取得し、未完了タスクを抽出する。

## 2. 各タスクの判定・処理
取得した各タスクを以下の基準で分類:

### A. KIRYUが即処理できるもの
- ファイル編集・コード修正・調査・ドキュメント更新
- Git操作（コミット・プッシュ）
- 設定ファイルの更新
→ その場で実行し、結果を記録

### B. サム（技術担当）に振るべきもの
- 新規の技術実装・サイト構築・API連携
- 複雑なデバッグ・パフォーマンス改善
→ サム用の指示書を生成（何を・なぜ・どこまで・期限）

### C. レオ（財務担当）に振るべきもの
- KPI集計・売上分析・経費処理
- 財務レポート作成
→ レオ用の指示書を生成

### D. NOYUTO判断が必要なもの
- 事業方針に関わる判断
- クライアント対応の方向性
- 予算の承認
→ 判断材料と選択肢を整理して報告に含める

## 3. 報告メールの作成・送信

以下の形式で報告を作成:

```
件名: [KIRYU巡回報告] YYYY-MM-DD(曜日) 深夜巡回

■ 処理済みタスク（KIRYUが実行完了）
- [タスク名]: [何をしたか]

■ サムへの指示書（技術実装）
- [タスク名]: [指示内容の要約]

■ レオへの指示書（財務・KPI）
- [タスク名]: [指示内容の要約]

■ NOYUTO判断待ち
- [タスク名]: [判断が必要な理由と選択肢]

■ 次回巡回までの注目事項
- [気づいた点・リスク・期限が近いもの]
```

Gmail MCPが使える場合:
mcp__gmail__authenticate で認証確認後、メール送信ツールで office.vonds@gmail.com に送信。

Gmail MCPが使えない場合:
報告内容を /home/ozawakiryu0902/projects/vonds/scripts/kiryu-patrol-reports/ ディレクトリに
YYYY-MM-DD_patrol-report.md として保存する。

## 制約
- 本番環境のファイル（.env、認証情報）は絶対に編集しない
- テストは /tmp/ 配下で行う
- 不明な点は「不明」と報告する。推測で断定しない
- 処理が5分以上かかりそうなタスクはスキップして報告に含める
