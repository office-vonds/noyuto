# 写メ日記メール自動投稿 技術設計書

## 概要

ポータルサイトの写メ日記を、メール投稿（gid@宛）で自動化する。
管理画面ログインやAPI不要。BANリスクゼロの最善ルート。

---

## なぜメール投稿か

| 方法 | BANリスク | 実装難度 | 運用安定性 |
|---|---|---|---|
| 管理画面を自動操作（Selenium等） | **高** — ボット検知でアカウントBAN | 中 | 低（サイト変更で壊れる） |
| 非公開APIを叩く | **高** — 利用規約違反 | 高 | 低 |
| **メール投稿（gid@宛）** | **ゼロ** — 正規機能 | 低 | **高** |

メール投稿はポータルサイトが公式に提供している機能。
人間がスマホからメールを送るのと全く同じ経路。検知不可能。

---

## システム構成

```
[コンテンツDB]
    │
    ├── キャスト別テンプレート（写真+文章パターン）
    │
    ▼
[投稿スケジューラー（Python + cron）]
    │
    ├── 投稿時間の最適化（ピーク時間帯に合わせる）
    ├── キャストごとの投稿頻度設定
    ├── 文章のバリエーション生成（テンプレート×ランダム要素）
    │
    ▼
[メール送信エンジン（smtplib）]
    │
    ├── 送信元: 各キャストの登録メールアドレス
    ├── 送信先: gid@（ポータルサイト指定のメール投稿アドレス）
    ├── 件名: 写メ日記のタイトル
    ├── 本文: 写メ日記の本文
    ├── 添付: 写真（JPEG）
    │
    ▼
[ポータルサイト]
    │
    └── 写メ日記として自動公開
```

#### 1-2. テスト用スクリプト作成
```python
# photo_diary_mailer.py（概要）
import smtplib
from email.mime.multipart import MIMEMultipart
from email.mime.text import MIMEText
from email.mime.image import MIMEImage

def send_photo_diary(
    cast_email: str,      # キャストの登録メールアドレス
    cast_password: str,    # メールアカウントのパスワード
    gid_address: str,      # ポータルサイトのgid@アドレス
    title: str,            # 写メ日記タイトル
    body: str,             # 写メ日記本文
    photo_path: str,       # 写真ファイルパス
):
    msg = MIMEMultipart()
    msg['From'] = cast_email
    msg['To'] = gid_address
    msg['Subject'] = title

    msg.attach(MIMEText(body, 'plain', 'utf-8'))

    with open(photo_path, 'rb') as f:
        img = MIMEImage(f.read())
        img.add_header('Content-Disposition', 'attachment', filename='photo.jpg')
        msg.attach(img)

    with smtplib.SMTP_SSL('smtp.gmail.com', 465) as server:
        server.login(cast_email, cast_password)
        server.send_message(msg)
```

#### 2-2. テンプレート設計方針
- 1キャストにつき最低20パターンの文章テンプレート
- テンプレートにランダム要素を埋め込む（日付、天気、時間帯、曜日）
- 同じ文章が連続しないようローテーション管理
- 口調はキャストごとにカスタマイズ（元気系、大人系、清楚系等）

#### 2-3. 写真管理
- 1キャストにつき最低50枚の写真ストック
- 写真の使用履歴を記録し、同じ写真の再利用間隔を管理
- 写真のバリエーション（衣装、角度、場所）を分類

POSTING_SCHEDULE = {

#### 3-2. 投稿頻度設定
| キャストランク | 投稿頻度/日 | 理由 |
|---|---|---|
| TOP10（売上上位） | 3〜4回 | PV最大化でさらに伸ばす |
| MID20（中間層） | 2〜3回 | 底上げしてTOP10に引き上げる |
| BOTTOM16（下位） | 1〜2回 | 最低限の露出確保 |

## セキュリティ・リスク管理

| リスク | 対策 |
|---|---|
| メールアカウントの漏洩 | .envファイルで管理、Gitにpushしない |
| 投稿内容の重複検知 | テンプレート×ランダム要素で毎回異なる文面を生成 |
| 連投によるスパム判定 | 投稿間隔を最低2時間以上空ける |
| 写真の重複使用 | 使用履歴DBで管理、最低2週間は再利用しない |
| ポータルサイトの仕様変更 | メール投稿は枯れた技術、変更リスク低 |
