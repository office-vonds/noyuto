# セッションログ: 2026-04-18 絆JOB FV画像改修

- 担当: サム
- ブランチ: main

## NOYUTOからの指摘

「各ページのFV画像がおかしい」

## 原因分析

2段階の問題が絡んでいた。

### 問題A（俺の Phase 1 で起こした事故）
前セッションで `place-recruit-images.py` 実行時、既存ページの先頭に
ヒーロー画像を挿入した際、以下の不具合を発生させた:
- contactで`<h2>まずはお気軽にご相談ください</h2>`の中にdiv挿入 → h2タグ破壊
- 既存FVがあるdormitory/guarantee等で画像重複
- salary FVに統計グラフ、contact FVにロゴなど内容不適合

### 問題B（元々の構造的問題）
既存FV画像が**姉妹店バニラや汎用素材の流用**だった:
| ページ | 流用元 |
|--------|-------|
| qa | vanilla-sub.jpeg |
| guarantee | vanilla-main.jpeg |
| security | vanilla-sub.jpeg |
| salary | official-hero.jpg |
| mature | hitoduma-main.jpg |
| beginner | vanilla-taiken.jpg |
| flow | official-hero.jpg + vanilla-interview.jpg + vanilla-sub4.jpeg |

絆JOBの求人サイトに、バニラ（別店舗）の画像が入っている状態 =
ブランドアイデンティティの毀損。NOYUTOが指摘したかったのはここ。

## 改修手順

### Step 1: Phase 1追加を全撤去
- `rollback-recruit-images.py` 作成 → 9ページ全件バックアップからHTML完全復元

### Step 2: FV画像を絆純正に置換
- `fix-fv-images.py` 作成
- 各ページの `src` 属性だけ書き換え、周辺構造・レイアウトは一切触らない
- バックアップ: `backups/fv_fix_20260418_2037/`

## 学んだこと（再発防止）

- **既存FVの構造を壊さず、`src` 属性だけ書き換えるのが最安全**（Phase 1の上積み挿入は悪手）
- 「img数が少ない」=「画像不足」ではない。**画像の質・ブランド整合**が先
- 「まずはお気軽」等の**自然文字列での置換は爆弾**。本文にも出現し構造破壊する
- 置換系オペレーションは `src` など限定属性の完全一致で行う
