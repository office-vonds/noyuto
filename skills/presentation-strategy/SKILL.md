# Presentation Strategy SKILL

プレゼンテーションの**構成・ストーリー設計**に特化。  
スライド作成（デザイン・技術）は `pptx/SKILL.md` を参照。

**使用タイミング:** pptxスキルでスライドを作る**前**に、構成を固める段階で使用。

-----

## Step 1: ブループリント（5項目を先に確定）

スライド作成に入る前に、以下を必ず定義する。

1. **目的** — このプレゼンで聴衆に何をさせたいか（1文）
2. **対象** — 誰に向けるか（役職・知識レベル・関心事）
3. **主要メッセージ** — 覚えて帰ってほしい1つのこと
4. **スライド枚数** — 目安（5分=5-7枚 / 15分=12-15枚 / 30分=20-25枚）
5. **CTA** — 最後に聴衆に取ってほしいアクション

-----

## Step 2: ナラティブ構造（5幕）

全スライドを以下の流れに配置する。

```
1. フック    — 注意を掴む（問い / 数字 / 意外な事実）
2. 問題提起  — 聴衆が共感する課題を描写
3. 洞察      — なぜこの問題が解けていないか（視点の転換）
4. 解決策    — 提案内容（具体的・実行可能）
5. 結論+CTA  — まとめ + 次のアクション
```

**ルール:**

- 各スライドは**1スライド=1メッセージ**
- フックは最初の10秒で聴衆を引き込む（質問形式が最も効果的）
- 「問題→解決」の落差が大きいほどプレゼンは刺さる

-----

## Step 3: 聴衆キャリブレーション

対象に合わせて以下を調整する。

| 対象    | 語彙レベル   | データ密度    | スライド密度  |
|---------|-------------|-------------|-------------|
| 経営層  | 平易・結論先行 | 低（KPIのみ）  | 少なめ       |
| 専門家  | 技術用語OK   | 高（根拠データ） | 多め         |
| 一般・顧客 | 日常語     | 最低限        | ビジュアル重視 |
| 投資家  | ビジネス用語  | 中（市場データ） | 簡潔         |

-----

## Step 4: スライド構成チェック

構成が固まったら、以下で検証。

- [ ] 最初の2枚で「なぜ聞くべきか」が伝わるか
- [ ] 各スライドのタイトルだけ読んでストーリーが通るか
- [ ] データスライドに「だから何？（So What?）」が書いてあるか
- [ ] CTAが具体的で実行可能か（「検討してください」は弱い）
- [ ] 枚数が時間に対して適切か

-----

## Step 5: エグゼクティブサマリー（必要時）

長いプレゼンでは冒頭 or 末尾に1枚のサマリースライドを追加。

含める要素:

- 結論（1文）
- 根拠（3点以内）
- 推奨アクション（1つ）
- 期待される成果（数字で）

-----
