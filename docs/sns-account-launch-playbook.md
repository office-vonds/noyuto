# SNSアカウント立ち上げ 1週間ルール（Ban回避プレイブック）

作成: 2026-04-17 / 責任: KIRYU
契機: @rina_xxxfree 凍結事故（2026-04-17）

> **絶対遵守**。このルールを飛ばしたら凍結リスクが一気に跳ね上がる。
> NOYUTOに同種作業を3回以上やらせたrina事故の教訓。

---

## 基本原則（CLAUDE.md 絶対遵守ルール #9 準拠）

1. 新規アカウントは最低3日間**手動投稿のみ**（自動化禁止）
2. 電話番号認証を必ず最初に通す
3. 自動フォロー・自動いいね・自動リツイートは**永久禁止**（engage.py 系はこの事故の直接原因）
4. Cookie方式移行時は**初回のみ Windows ChromeからWeb UIで投稿**し、2日以上空けてから自動化cron登録
5. WSL/サーバーIPから直接Cookie認証すると新デバイス判定で凍結リスク増大
6. 投稿頻度上限: 新規2投稿/日・安定期3投稿/日

## Day 0（準備日・所要60分）

**場所: Windows Chrome (NOYUTOメインPC)**

- [ ] アカウント作成（x.com/signup）
- [ ] メールアドレス認証
- [ ] **電話番号認証（必須）**
- [ ] プロフィール設定（名前・自己紹介・アイコン・ヘッダー）
- [ ] 初期フォロー: 5-10人（同ジャンルのインフルエンサー）
- [ ] 固定ツイート1本（ペルソナの自己紹介）

**記録**: KIRYUは `state/sns-accounts.md` に登録情報・作成日・使用IP・認証履歴を記録

---

## Day 1-3（手動投稿期・所要各日5-10分）

**場所: Windows Chrome（WSL/Linuxから触らない）**

- [ ] 朝8:00 手動投稿1本（ストックから選んで手コピペ）
- [ ] 夜20:00 手動投稿1本
- [ ] いいね・リツイート・コメントは自然に（**1日10件以下・フォロー5人以下**）

**やってはいけないこと:**
- cronでの自動投稿
- Cookie方式への移行
- engage.py 類の起動
- 1日3投稿以上
- フォロー10人/日以上

---

## Day 4-6（移行準備期・所要各日10-15分）

**場所: Windows Chrome 継続**

- [ ] 手動投稿継続（朝夜2回）
- [ ] 初期フォロワー返信に手動対応
- [ ] Day 6に Cookie 取得の準備
  - Windows Chromeで https://x.com にログインした状態を維持
  - F12 → Application → Cookies → auth_token, ct0 の値を**この時点ではまだコピーしない**

---

## Day 7-（半自動化移行・慎重導入）

- [ ] Day 7: Windows Chrome から Cookie 取得 → **Win上で** `node x-cookie-post.js --setup` 実行
  - 可能ならWSL ではなく Windows Node.js を使う（IPを同一に保つ）
  - WSLを使う場合は初回は必ず dry-run でテスト、本番投稿は手動確認
- [ ] Day 7〜13: 1日1回の自動投稿のみ（cron化）
- [ ] Day 14 以降: 朝夜2回に増やす
- [ ] Day 30 以降: engage.py 類の自動化は**採用しない**（手動エンゲージメント維持）

---

## 継続運用（Month 2 以降）

- 週1回の Cookie 健康診断 cron（401検知時に Slack/メール通知）
- Cookie期限切れは早期検知（放置で Ban 誘発）
- 投稿内容は **スパム判定されない自然な文体**を維持
- プロフィール・固定ツイートの定期見直し
